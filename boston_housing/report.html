编程练习2：定义衡量标准¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数 $R^2$ 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

$R^2$ 的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的 $R^2$ 值为0还不如直接用平均值来预测效果好；而一个 $R^2$ 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。模型也可能出现负值的 $R^2$，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

在下方代码的 performance_metric 函数中，你要实现：

  • 使用 sklearn.metrics 中的 r2_score 来计算 y_true 和 y_predict 的 $R^2$ 值，作为对其表现的评判。
  • 将他们的表现评分储存到 score 变量中。

将数据集按一定比例分为训练用的数据集和测试用的数据集对学习算法有什么好处？

如果用模型已经见过的数据，例如部分训练集数据进行测试，又有什么坏处？

问题 3 - 回答:¶

训练集用于训练模型，得到合适的模型参数。 测试集用于测试模型参数对现实世界的表现，或者说对新样本的预测能力。例如通过测试集我们可以检查模型是否出现欠拟合或者过拟合的问题。

非常重要的一点需要注意：
测试集一定一定一定不能参与训练。在整个模型训练过程中，测试集必须隐藏起来，直到训练结束得到我们自认为正确的模型参数，这个时候才能够使用测试集，也就是说测试集肯定是出现在整个训练过程中的最后。

如果测试集参与了训练，它就失去了意义。就好像考试之前你拿到了试题，最终的考试分数并不能体现你的能力，最多也就体现你的记忆力。

提示：在解释k-fold交叉验证时，一定要理解'k'是什么，和数据集是如何分成不同的部分来进行训练和测试的，以及基于'k'值运行的次数。 在考虑k-fold交叉验证如何帮助网格搜索时，你可以使用特定的数据子集来进行训练与测试有什么缺点，以及K折交叉验证是如何帮助缓解这个问题。

问题 8 - 回答：¶

  • 什么是K折交叉验证法？
    • 数据分割时，我们将全部数据集分为训练集和测试集两部分。
    • 将上述训练集分为K份，在机器学习领域称K份为K包，编号为1,2 ... K-1,K，依次取第N包作为为验证集，其他K-1包作为训练集进行K次训练，取K次训练得分的平均值为目标得分。
    • 通过K次训练保证最终整个训练集都参与了训练。
  • GridSearchCV 是如何结合交叉验证来完成对最佳参数组合的选择的？
    • 在网格搜索法中，对每个模型参数的取值均进行一次K折交叉验证，设有M个模型参数，每个模型参数取值有N个，则最终模型经过的训练次数为'M*N*K'
    • K折样本划分可能存在差别，因此为了减少样本划分带来的差别，再一次K折检查验证中可能进行P次样本划分，因此最终模型经过的训练次数也可能为'M*N*K*P'
  • GridSearchCV 中的'cvresults'属性能告诉我们什么？
    • 保存着模型训练过程尝试的所有参数组合和对应的分数。
  • 网格搜索为什么要使用K折交叉验证？K折交叉验证能够避免什么问题？
    • 简单交叉验证，是将数据分为两部分，一部分用于训练一部分用于验证，计算成本低，但是无法充分利用数据。
    • 留一交叉验证（leave-one-out cross validation）可以完全充分利用数据，可以在数据缺乏的情况下使用，但是数据躲起来后，计算成本很大
    • K折交叉验证则结合上述两种方法的优点，相比简单交叉验证，利用数据更加充分，相比留一交叉验证，计算成本低。

def fit_model(X, y):

    # Create cross-validation sets from the training data
